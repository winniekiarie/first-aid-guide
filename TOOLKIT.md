## 1. Title & Objective

**Project Title:** First Aid Step-by-Step Guide – Powered by the Anthropic Claude API

**Technology Chosen:** Anthropic Claude API (`claude-sonnet-4-20250514`)

**Why I chose it:**
The Anthropic Claude API is one of the most accessible and powerful generative AI APIs available today. It requires zero machine learning knowledge to use — you simply send a text message and receive an intelligent response. For a first aid guide, this is ideal because:
- First aid instructions need to be clear, calm, and structured — exactly what Claude excels at
- The API can handle any emergency description, not just a fixed list of scenarios
- It's beginner-friendly: a single API call with plain JSON is all that's needed

**End Goal:**
Build a working web application where users can describe any medical emergency (or pick from 12 quick scenarios) and receive clear, numbered, step-by-step first aid instructions generated by Claude AI — with no setup beyond opening a browser.

## 2. Quick Summary of the Technology

### What is the Anthropic Claude API?

The Anthropic Claude API is a RESTful API that gives developers access to Claude — Anthropic's family of large language models (LLMs). You send it a text message (called a "prompt"), and it returns a human-quality text response.

### Where is it used?

- Chatbots and virtual assistants
- Document summarization tools
- Code assistants and debugging helpers
- Educational tutoring platforms
- Health information apps (like this project)

### One Real-World Example

**Notion AI** (the popular note-taking app) uses large language models via APIs like Claude's to let users rewrite notes, generate summaries, and translate documents — all within their existing product, without building an AI model from scratch.

## 5. Minimal Working Example

### What the Example Does

This is the core of the project — a single JavaScript `fetch()` call to the Anthropic API that sends a first aid question and prints the response.

### Code (Standalone HTML – No Build Needed)

```html
<!DOCTYPE html>
<html>
<head><title>Claude API – Hello World</title></head>
<body>
  <h2>First Aid AI Test</h2>
  <button onclick="askClaude()">Ask Claude: How to treat a burn?</button>
  <pre id="output">Response will appear here...</pre>

  <script>
    async function askClaude() {
      const API_KEY = 'sk-ant-YOUR-KEY-HERE'; // Replace with your key

      const response = await fetch('https://api.anthropic.com/v1/messages', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'x-api-key': API_KEY,
          'anthropic-version': '2023-06-01',
          // Required for direct browser access (no backend server)
          'anthropic-dangerous-direct-browser-access': 'true'
        },
        body: JSON.stringify({
          model: 'claude-sonnet-4-20250514', // The AI model to use
          max_tokens: 1000,                  // Max length of response
          messages: [
            {
              role: 'user',
              content: 'How do I treat a minor burn? Give me step-by-step instructions.'
            }
          ]
        })
      });

      const data = await response.json();

      // The text response is in data.content[0].text
      document.getElementById('output').textContent = data.content[0].text;
    }
  </script>
</body>
</html>
```

### Expected Output

```
1. Cool the burn immediately under cool (not cold) running water for 10–20 minutes.
2. Do not use ice, butter, or toothpaste — these can cause further damage.
3. Remove any jewelry or tight items near the burned area (before swelling starts).
4. Cover loosely with a sterile non-stick bandage or clean cloth.
5. Take over-the-counter pain relief (ibuprofen or paracetamol) if needed.
6. Seek medical attention if the burn is larger than 3 inches, on the face/hands/joints,
   or appears white/charred (indicating a serious burn).
```

## 6. AI Prompt Journal

### Prompt 1 – Learning the API Basics

**Prompt used:**
> "Give me a beginner-friendly introduction to the Anthropic Claude API. What is it, how does it work, and how do I make my first API call in JavaScript?"

**AI Response Summary:**
The AI explained that Claude is a REST API that accepts a JSON body with a `model`, `max_tokens`, and `messages` array. It provided a full working example in plain JavaScript using `fetch()`, explained what each parameter does, and showed how to extract the text response from `data.content[0].text`.

**Evaluation:** Extremely helpful. It saved hours of reading documentation by giving a runnable example immediately. The explanation of the `messages` array format (role + content) was particularly clear.

---

### Prompt 2 – Designing the System Prompt

**Prompt used:**
> "I'm building a first aid guide app using the Claude API. Write me a system prompt that will make Claude respond with calm, numbered step-by-step first aid instructions suitable for someone in a stressful emergency situation."

**AI Response Summary:**
Claude suggested a system prompt that instructs it to: (1) state when to call 911, (2) use numbered steps, (3) use simple language, (4) include warnings about what NOT to do, and (5) end with a recovery note. It also recommended keeping language calm and direct.

**Evaluation:** Very useful. The suggested system prompt became the exact one used in the final app with minimal edits.

---

### Prompt 3 – Handling the Browser CORS Issue

**Prompt used:**
> "I'm calling the Anthropic API directly from a browser using fetch() and getting a CORS error. How do I fix this without setting up a backend server?"

**AI Response Summary:**
The AI explained that Anthropic supports direct browser access by adding the header `'anthropic-dangerous-direct-browser-access': 'true'`. It warned this should only be used for development/demos (never production) since it exposes the API key in the browser. It also suggested using a backend proxy for production apps.

**Evaluation:** This unblocked a critical issue in under 2 minutes. Without this, the app wouldn't work at all in a browser.

---

### Prompt 4 – Building the UI

**Prompt used:**
> "Give me a clean, beginner-friendly HTML + CSS layout for a first aid app with: a header, a grid of scenario buttons (burns, choking, CPR etc.), a text input area, and a response card. No external CSS frameworks."

**AI Response Summary:**
Claude provided a full CSS layout using CSS Grid for the scenario buttons, Flexbox for the input row, and styled cards for the response area. It included color variables and hover effects.

**Evaluation:** Saved significant time. The grid layout for scenario buttons worked perfectly. Minor color adjustments were made to match a medical red/green theme.

---

### Prompt 5 – Parsing and Formatting the Response

**Prompt used:**
> "The Claude API returns text with numbered steps like '1. Cool the burn...' How can I parse this in React and render each step with a styled numbered circle instead of plain text?"

**AI Response Summary:**
The AI provided a `parseResponse()` function that splits the text by newlines, uses regex to detect numbered steps (`/^\d+[\.\)]/`), and returns an array of typed blocks (step, warning, tip, paragraph). It also suggested handling warning lines that start with ⚠️ and tip lines that start with ✅.

**Evaluation:** This became the exact `parseResponse` function in the final app. Very practical.

## 7. Common Issues & Fixes

### Issue 1 – CORS Error in Browser

**Error:**
```
Access to fetch at 'https://api.anthropic.com/v1/messages' from origin 
'null' has been blocked by CORS policy
```

**Fix:**
Add this header to your fetch request:
```javascript
'anthropic-dangerous-direct-browser-access': 'true'
```

**Reference:** [Anthropic Docs – Client-side access](https://docs.anthropic.com/en/api/getting-started)

---

### Issue 2 – 401 Unauthorized

**Error:**
```json
{ "error": { "type": "authentication_error", "message": "invalid x-api-key" } }
```

**Fix:**
- Double-check your API key is correct (starts with `sk-ant-`)
- Make sure you're passing it in the `x-api-key` header, not `Authorization`
- Check for accidental spaces before/after the key

---

### Issue 3 – 429 Rate Limit Exceeded

**Error:**
```json
{ "error": { "type": "rate_limit_error", "message": "rate limit exceeded" } }
```

**Fix:**
- Wait 60 seconds and try again
- Free tier has usage limits; check your dashboard at console.anthropic.com
- Add a loading state to prevent users from submitting multiple requests at once

---

### Issue 4 – Response Cuts Off Mid-Sentence

**Symptom:** Claude's response ends abruptly in the middle of a step.

**Fix:** Increase `max_tokens`:
```javascript
max_tokens: 2000  // increase from 1000
```

---

### Issue 5 – `data.content` is Undefined

**Symptom:** `TypeError: Cannot read properties of undefined (reading '0')`

**Fix:** Always check for errors first:
```javascript
if (!res.ok) {
  const err = await res.json();
  throw new Error(err.error?.message || 'API Error');
}
const data = await res.json();
const text = data.content?.[0]?.text || 'No response.';
```

---

### Issue 6 – App Works Locally but Not After Opening HTML Directly

**Symptom:** Blank page or React errors when opening `index.html` from file system (file:// URL)

**Fix:** Run a local server instead:
```bash
python3 -m http.server 8080
# Visit http://localhost:8080
```

## 📝 Learning Reflections

**What surprised me most:**
The Claude API is dramatically simpler than expected. There's no SDK required — a single `fetch()` call with a JSON body is all it takes to get intelligent, well-structured responses. The hardest part was the CORS header for browser access, which AI helped me solve in seconds.

**How AI improved my productivity:**
Using Claude to learn Claude was uniquely effective. I could ask "why isn't this working?" and paste my code, and get a precise fix rather than spending 20 minutes on Stack Overflow. The system prompt design prompt alone saved hours of trial and error.

**What I'd do differently:**
In production, I'd add a simple backend proxy (Node.js/Express or Python/Flask) to avoid exposing the API key in the browser. I'd also add streaming support so the response appears word-by-word rather than all at once.
